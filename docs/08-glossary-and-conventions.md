# Glossary and conventions

## Glossary

| Term | Definition |
|---|---|
| **Business** | A tenant workspace. One user can belong to many businesses. Stored in `businesses` table. |
| **Workspace** | Synonym for **business** in the UI. |
| **Membership** | A `(user, business, role, status)` tuple. The composite primary key is `(userId, businessId)` (`app/api/src/db/schema.ts:230`). |
| **Role** | One of `Owner`, `Admin`, `Manager`, `Staff`, `Viewer`. Ranked by `workspace/permissions.ts:11`. |
| **Invitation** | Email-bound token that grants a membership when accepted. Expires after 7 days. |
| **Dograh** | The self-hosted voice-agent platform included as a git submodule. |
| **Workflow** | A Dograh call graph (nodes + edges) that defines greeting, conversation, closing, and interrupt behaviour. |
| **Embed token** | A public token issued by Dograh that allows a browser widget to start a call for a specific workflow. |
| **Knowledge** | Tenant-scoped documents, text, or website references used as context by the voice agent. |
| **Outbox** | The `outbox_events` table used as a persistent async job queue. |
| **Sync** | Short for synchronizing a Vocalonix business's desired state into a Dograh workflow. |
| **MVP lab** | The unprotected `/secret/*` routes used for testing Dograh without authentication. |
| **Magic link** | One-time sign-in link generated by `magicLinkRequests` and sent by email. |
| **better-auth** | The authentication library used for sessions, password, and magic-link flows. |
| **Eden Treaty** | The generated type-safe API client used by the frontend (`app/web/src/api.ts:17`). |
| **Tenant** | A business and its isolated settings, knowledge, and Dograh workflow mapping. |
| **Widget snippet** | The `<script>` tag a customer pastes into their own website to load the browser-call widget. |

## Code conventions

### Naming

- Files are lower-case; components are PascalCase. Example: `app/web/src/components/ui/Button.tsx` exports `Button`.
- API route files are plural or resource-named: `auth/routes.ts`, `workspace/routes.ts`, `tenant/routes.ts`.
- Tests live next to the file they test: `app/api/src/outbox.test.ts` tests `app/api/src/outbox.ts`.

### Imports

- Absolute imports are used within packages. The web imports `api.ts` from `"./api"` and `AuthProvider` from `"./auth/AuthProvider"`.
- The API does not use path aliases; it uses relative paths like `"../db/client"`.

### Error handling

- API routes throw `ApiError` or `DograhError`. The global handler in `app/api/src/index.ts:94` converts them to JSON responses.
- Frontend API calls are wrapped by `unwrap()` in `app/web/src/api.ts:50`, which throws `ApiClientError` with `status` and `code`.
- React components catch errors and display `value instanceof Error ? value.message : "..."`.

### Type safety

- Both API and web use `strict` TypeScript and `noUncheckedIndexedAccess`.
- The API uses `t.*` validators from Elysia for route bodies.
- The web uses Zod schemas + `zodResolver` for forms and `react-hook-form`.

### Database conventions

- Primary keys are `text("id")` with `randomUUID()` generated by the application.
- Timestamps use `timestamp({ withTimezone: true })` and default to `now()`.
- Soft deletes use `deletedAt` timestamp (e.g. `businesses.deletedAt`, `businessKnowledge.deletedAt`).
- JSONB is used for arrays and maps (`businessHours`, `completedSteps`, `allowedDomains`).
- Audit logs are append-only with `action`, `targetType`, `targetId`, and `payload` JSONB.

### Environment conventions

- All runtime configuration is in `app/api/src/env.ts` and validated at startup.
- Secrets are read from environment variables only. Never commit `.env`.
- `VITE_*` variables are baked into the web bundle at build time.

## Testing conventions

- Tests use `bun:test`.
- Unit tests are colocated with the module they test.
- Run all tests with `bun run test` from the workspace root.
- CI runs `bun install --frozen-lockfile`, `./scripts/setup.sh`, `bun run db:migrate`, `bun run typecheck`, `bun run test`, `bun run build`.

To add a new test, create a file named `<module>.test.ts` next to the module and import from `bun:test`.
